# S6 — 리드→파트너 퍼널·운영 알림 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** admin이 파트너 요청 상태를 직접 전이(`contacted` 도입)할 수 있게 하고, 운영 필수 이벤트 4종(리드·고위험 케이스·문서 업로드·파트너 요청)에 `sendOpsAlert`를 배선한다.

**Architecture:** 정찰 결과 — `/admin/leads`는 이미 존재(클라 이중 게이트+API requireAdmin, 의도된 패턴, 손대지 않음). pending→matched(admin 배정)·matched→accepted/closed(파트너 계정) 전이는 기구현. 미구현: (T1) admin 직접 전이 `PATCH /api/partner-requests/[id]`(+`contacted` 상태 — plain string이라 스키마 마이그레이션 불필요, 주석·허용값만) + 최소 UI 버튼 + `tx.auditEvent.create` 감사, (T2) `sendOpsAlert` 4곳 배선(fire-and-forget). 두 태스크 파일 분리 → 병렬.

**Tech Stack:** 기존 `requireAdmin`(src/lib/api/security.ts:109), `assignment.ts`의 auditEvent 패턴, `sendOpsAlert`(src/lib/ops/alerts.ts:199 — env `OPS_ALERT_SLACK_WEBHOOK_URL` 미설정 시 no-op성 결과), `parseJsonBody`+zod.

## Global Constraints

- **스키마 마이그레이션 금지** — `PartnerRequest.status`는 plain string. 코드 레벨 허용값·주석만. 원격 DB 작업 없음.
- **ops 알림은 best-effort**: 반드시 fire-and-forget(`.catch(console.warn)`) 또는 try/catch — 실패가 원 요청(리드 저장·케이스 생성·업로드·파트너 요청)을 절대 깨지 않게. **DB 트랜잭션 내부에서 호출 금지**(커밋 후).
- **알림 페이로드에 PII 금지**: contact/이메일/nickname/질문 원문 넣지 말 것. id·category·pathKey·상태·개수 수준만.
- 기존 전이 로직(`assignPartnerRequest`/`updatePartnerRequestStatus`)·파트너 계정 경로 무변경.
- 기존 인앱 알림(notifyUsers) 유지 — ops 알림은 추가.
- 새 API는 `requireAdmin(roles: ["owner","admin"])` + `parseJsonBody`+zod. 상태 전이는 허용맵 강제(임의 값 거부 400).
- 커밋은 slice/s6-funnel에서만, push 금지. push 직전 재fetch·rebase.
- 검증: 전이 허용맵 단위 테스트(`scripts/test-*` 컨벤션, ci 배선) + tsc/eslint/build.

- [ ] **Step 1: 실패 테스트 먼저** — `scripts/test-partner-request-transitions.ts`: node:assert로 허용맵 검증(pending→contacted true, pending→accepted false, closed→contacted false, contacted→closed true, matched→closed true, 동일상태 false, 미지 상태 false). 실행 → FAIL(모듈 없음) 확인.
- [ ] **Step 2: 허용맵 구현** — `src/lib/partners/status-transitions.ts`에 순수 함수 + `ADMIN_SETTABLE_STATUSES = ["contacted","closed"] as const`. 테스트 PASS.
- [ ] **Step 3: 라우트** — `PATCH /api/partner-requests/[id]`:

  - `parseJsonBody` + zod `{ status: z.enum(["contacted","closed"]) }`.
