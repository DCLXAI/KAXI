# S3 — AI 채팅 웜다크·고양이 아바타·에스컬레이션 UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** consult/agent 채팅을 확정 아이덴티티(웜다크)로 전환하고 KaxiCat 아바타를 적용하며, needsHumanExpert 시 케이스 생성 결과를 사용자에게 보여준다(로그인=확인 카드/비로그인=로그인 CTA).

**Architecture:** 정찰 결과 **백엔드는 운영자 선작업 완료** — consult/agent/diagnosis 라우트 모두 `currentAuthenticatedStudentProfileId()`(세션 파생·role 검증) → `maybeCreateHighRiskEscalationCase()`(dedupe·타임라인·감사·알림 트랜잭션) 배선됨. 잔여는 (1) `.chat-surface` 웜다크 적용(사용처 0), (2) 채팅 아바타(현재 Scale/Bot 아이콘), (3) 케이스 생성 결과를 응답에 실어(현재 fire-and-forget이라 UI가 모름) 카드/CTA 분기. T1(리스킨)과 T2(에스컬레이션 UX)가 같은 파일을 만지므로 **순차 실행**.

**Tech Stack:** `.chat-surface`+`.dark` 클래스(globals.css:83 `.dark,.chat-surface` 변수 블록), `<KaxiCat state="breath" inverted>`, 기존 인라인 4로케일 삼항 컨벤션.

## Global Constraints

- **핵심 함정**: `@custom-variant dark (&:is(.dark *))`는 리터럴 `.dark`만 매칭 → `.chat-surface`만 붙이면 시맨틱 토큰은 웜다크로 바뀌지만 `dark:` 프리픽스 클래스(AgentResponseCard:110, AgentClarifyPanel:62-114)는 발화 안 함. 반드시 **`chat-surface dark` 병기**.
- 채팅 로직(useConsultChat/useAgentChat의 fetch·history·mode·세션, 라우트의 LLM/RAG/검증) 무변경 — T2의 응답 필드 추가·await 전환만 예외(명시된 범위).
- LLM/RAG 파이프라인·rate limit·`maybeCreateHighRiskEscalationCase` 내부·알림 시스템 변경 금지.
- 다크 배경 위 KaxiCat은 `inverted` 필수. 하드코딩 색 신규 금지(기존 amber/blue 박스는 dark: 변형 보강만).
- 신규 문구는 파일 기존 컨벤션(인라인 로케일 삼항) 그대로 ko/vi/mn/en 4개.
- duration ≤300ms, `transition-all` 금지, 신규 모션은 reduced-motion 존중(기존 Tier1~3 기준 유지).
- 커밋은 slice/s3-chat에서만, push 금지. push 직전 재fetch·rebase.
- 시각 검증: Playwright(consult 랜딩+대화 상태 웜다크, ko/vi, 데스크톱+모바일) → 사용자 검수.
