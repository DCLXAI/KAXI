# S4 — 로그인·온보딩 슬라이스 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** 로그인·온보딩 화면(UnifiedAuthForm + /login 래퍼)에 확정 아이덴티티를 적용한다.

**Architecture:** i18n·다국어는 **이미 완료됨**(운영자 선작업: `AUTH_COPY` ko/vi/mn/en 완비, `CALLBACK_ERRORS` 4로케일, `syncUser`가 `locale: lang` 전달, searchParams에서 lang 읽음). 리다이렉트도 로케일 유지됨. 따라서 S4는 **브랜드 리스킨만**: 아이보리 에디토리얼, `font-serif` 타이틀, `<KaxiCat>` 환영, 브랜드 토큰. 로직·AUTH_COPY·인증 흐름 무변경.

**Tech Stack:** Next.js 16, Tailwind 브랜드 토큰, `<KaxiCat>`, shadcn.

## Global Constraints

- 스펙 S4 행이지만 i18n/완결성은 이미 해소 — 확인만. 실작업은 리스킨.
- 인증 로직(syncUser·router.replace·callback·supabase 호출)·AUTH_COPY 딕셔너리·CALLBACK_ERRORS **변경 금지**. 스타일 클래스+KaxiCat만.
- 브랜드 토큰(globals.css) 변경 금지. 하드코딩 색 금지(토큰 클래스). 신규 문구 최소화(필요시 AUTH_COPY에 4로케일 추가 — next-intl 아님, 기존 로컬 딕셔너리 방식 유지).
- 브랜드 아이덴티티: `bg-background`/`text-foreground`/`bg-primary`/`bg-card`, `font-serif`(타이틀), `<KaxiCat state size inverted>`(`@/components/brand/KaxiCat`).
- 커밋은 slice/s4-auth에서만, push 금지. origin/main 이동 → push직전 rebase.
- 시각 검증: 배포 전 Playwright 스크린샷(로그인 데스크톱+모바일, 3개 언어 중 vi 하나 포함) → 사용자 검수.

### Task 1: 로그인·온보딩 에디토리얼 리스킨

**Files:**
- Modify: `src/components/auth/UnifiedAuthForm.tsx`, `src/app/login/page.tsx`(래퍼 레이아웃)

**Interfaces:**
- Consumes: 기존 `AUTH_COPY[lang]`, `<KaxiCat>`, 브랜드 토큰.

- [ ] **Step 1: 로그인 카드 리스킨** — `/login` 페이지 래퍼와 UnifiedAuthForm를 아이보리 에디토리얼로: 중앙 정렬 카드(`bg-card` 위 `bg-background`), 타이틀(`AUTH_COPY[lang].title`)에 `font-serif`, 서브타이틀 `text-muted-foreground`, 입력 필드·버튼을 브랜드 토큰 클래스로(주 CTA는 `bg-primary`). 탭(로그인/회원가입/이메일링크/재설정)이 있으면 차분한 스타일.

- [ ] **Step 2: KaxiCat 환영** — 로그인 카드 상단(타이틀 위)에 `<KaxiCat state="happy" size={56} label={AUTH_COPY[lang].title}/>` 또는 breath 상태로 환영 포인트. 다크 배경 아니므로 inverted 미사용.

- [ ] **Step 3: 하드코딩 색·잔여 삼항 확인** — UnifiedAuthForm에 인라인 색(hex/arbitrary)이나 `lang === ` 삼항이 있으면 토큰/AUTH_COPY로. `grep -nP "#[0-9a-fA-F]{3,6}|lang === " src/components/auth/UnifiedAuthForm.tsx` 잔존 확인. AUTH_COPY에 새 키 추가 시 ko/vi/mn/en 4개 다.

- [ ] **Step 4: 검증** — `bunx tsc --noEmit` + `bunx eslint src/components/auth/UnifiedAuthForm.tsx src/app/login/page.tsx` + `bun run build` 클린. 인증 로직 diff에 없음 확인(className+KaxiCat만). 가능하면 Playwright로 `/login?lang=ko`·`/login?lang=vi` 스크린샷 자가검증.

- [ ] **Step 5: Commit** — `git add src/components/auth/UnifiedAuthForm.tsx src/app/login/page.tsx && git commit -m "feat(auth): editorial reskin of login/onboarding with KaxiCat welcome"`
