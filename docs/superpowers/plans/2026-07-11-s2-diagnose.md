# S2 — 진단 슬라이스 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. 체크박스(`- [ ]`) 단위 실행.

**Goal:** 진단 플로우(폼 위저드·결과)에 확정 아이덴티티를 적용하고, API를 zod로 검증하며, 결과→상담 후속 연결과 비로그인 가입 유도를 추가한다.

**Architecture:** 멀티스텝 위저드·제출 로딩(`submitting`)은 **이미 구현됨**(운영자 선작업) — 리스킨만. `/api/diagnosis`의 손수 검증을 기존 `parseJsonBody`(`@/lib/api/validation`, 앞서 landed) + zod 스키마로 교체. 결과 화면(DiagnosisResult)에 상담 CTA·가입 유도 추가 + 삼항 9곳 tr() 정리.

**Tech Stack:** Next.js 16, zod v4(parseJsonBody 패턴), Tailwind 브랜드 토큰, `<KaxiCat>`, 기존 tr() i18n.

### Task 1: /api/diagnosis zod 검증

**Files:**
- Modify: `src/app/api/diagnosis/route.ts`
- Test: `scripts/test-diagnosis-rules.ts`(기존, 있으면 검증 케이스 추가) 또는 신규 `scripts/test-diagnosis-validation.ts` + package.json 배선

**Interfaces:**
- Consumes: `parseJsonBody(request, schema, {maxBytes})`(`@/lib/api/validation`) — leads에서 확립. zod v4.

- [ ] **Step 1: 현재 route.ts 검증부 파악** — `req.json().catch()`(112줄 부근) + 손수 `asString/asNumber/asBoolean/isOneOf` 헬퍼로 body를 파싱하는 부분. 진단 입력 필드(goal/koreanLevel/budget/region/usingBroker/hasHistory/nationality/age 등 — recommendPath 입력)를 파악.

- [ ] **Step 2: zod 스키마 작성** — route.ts 상단에 `diagnosisSchema = z.object({...})`. 각 필드를 기존 손수 검증과 **동등하게**: enum 필드는 `z.enum([...])`(기존 isOneOf 값 목록), 숫자는 `z.coerce.number().min(0)`(기존 asNumber가 문자열 "6,000,000"도 허용했다면 `z.preprocess`로 콤마 제거 후 coerce), boolean은 `z.coerce.boolean()`, 문자열은 `z.string().max(80)`. **기존 허용 입력을 깨지 않도록** 손수 헬퍼의 동작을 그대로 스키마로 옮길 것(특히 콤마 포함 숫자).

- [ ] **Step 3: POST 재작성** — `const parsed = await parseJsonBody(req, diagnosisSchema); if (!parsed.ok) return parsed.response; const input = parsed.data;` 이후 `recommendPath(input)` 등 기존 로직 그대로. 손수 헬퍼(asString 등)가 이 라우트에서만 쓰이면 제거, 공용이면 유지.

- [ ] **Step 4: 테스트** — 유효 body→200(기존 응답 형태 유지), 콤마 숫자("6,000,000")→정상 파싱, 필수 필드 누락/잘못된 enum→400. RED→GREEN. 기존 `test:diagnosis`(test-diagnosis-rules.ts)가 이 라우트를 부르면 회귀 없는지 실행. 신규 스크립트면 ci 배선.

- [ ] **Step 5: Commit** — `git add src/app/api/diagnosis/route.ts scripts package.json && git commit -m "fix(diagnosis): validate API input with zod (parseJsonBody)"`
