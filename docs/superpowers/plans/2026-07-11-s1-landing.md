# S1 — 랜딩·헤더·푸터 슬라이스 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 방문자 첫 화면(랜딩·헤더·푸터)에 확정 아이덴티티(아이보리 에디토리얼 + 세리프 + 고양이)를 적용하고, 완결성 버그(features 키 중복, 채팅에서 푸터 숨김, 회사정보 부재)와 하드코딩 문구를 고친다.

**Architecture:** Phase 0의 브랜드 토큰·`<KaxiCat>`·폰트 인프라 위에서 진행. 한글 헤드라인 세리프는 Phase 0에서 미룬 self-host 폰트(Nanum Myeongjo)를 추가해 `--font-serif` 스택에 얹는다. i18n은 기존 `tr()`+`useLangStore` 체계 유지 — 버그·하드코딩만 수정(전면 next-intl 이관은 범위 밖).

**Tech Stack:** Next.js 16, Tailwind v4(브랜드 토큰), next/font/local, 기존 `tr()` i18n, shadcn.

## Global Constraints

- 스펙: `docs/superpowers/specs/2026-07-11-launch-readiness-redesign-design.md` S1 행.
- **i18n 범위(사용자 확정)**: features 키 중복·하드코딩 한국어만 수정, `tr()`+`useLangStore` 체계 유지. Landing/Header를 next-intl로 이관하지 않는다.
- 브랜드 토큰(globals.css)·`<KaxiCat>`·팔레트는 Phase 0에서 확정 — 값 변경 금지. 새 문구는 4개 로케일(ko/vi/mn/en) 모두 `translations.ts`에 추가(i18n-parity 가드가 messages/*.json만 검사하므로 tr() 사전은 수동 4로케일).
- 신규 문자열은 tr() 키로. 인라인 하드코딩 한국어 금지.
- 시각 검증: 배포 전 Playwright 스크린샷(데스크톱+모바일 390px) → 사용자 확인 루프.
- 커밋은 작업 브랜치에서만, push 금지(오케스트레이터가 랜딩). origin/main 활발히 이동 → push 직전 rebase.

### Task 1: 한글 헤드라인 세리프 self-host (Phase 0 이월)

**Files:**
- Create: `src/fonts/NanumMyeongjo-{Regular,Bold,ExtraBold}.woff2`
- Modify: `src/app/fonts.ts`, `src/app/globals.css`(`--font-serif` 스택)

**Interfaces:**
- Produces: `--font-serif` 스택에 한글 커버 세리프 포함 → `font-serif` 클래스가 한글 제목도 세리프로 렌더.

- [ ] **Step 1: Nanum Myeongjo woff2 다운로드** (fontsource, SIL OFL)
```bash
mkdir -p src/fonts
for w in 400:Regular 700:Bold 800:ExtraBold; do
  wght=${w%%:*}; name=${w##*:}
  curl -fL -o "src/fonts/NanumMyeongjo-${name}.woff2" \
    "https://cdn.jsdelivr.net/fontsource/fonts/nanum-myeongjo@latest/korean-${wght}-normal.woff2"
done
ls -la src/fonts/NanumMyeongjo-*.woff2   # 3개, 각 수백KB
```
(다운로드 실패 시 BLOCKED 보고 — URL 형식이 바뀌었을 수 있으니 `https://api.fontsource.org/v1/fonts/nanum-myeongjo`로 실제 파일 경로 확인 후 재시도.)

- [ ] **Step 2: `fonts.ts`에 localFont 추가**
```ts
export const nanumMyeongjo = localFont({
  src: [
    { path: "../fonts/NanumMyeongjo-Regular.woff2", weight: "400" },
    { path: "../fonts/NanumMyeongjo-Bold.woff2", weight: "700" },
    { path: "../fonts/NanumMyeongjo-ExtraBold.woff2", weight: "800" },
  ],
  display: "swap",
  variable: "--font-serif-ko",
});
```
`layout.tsx`의 `<html>` className에 `${nanumMyeongjo.variable}` 추가.

- [ ] **Step 3: `globals.css` `--font-serif` 스택 갱신** — `@theme inline`의 serif 줄:
```css
  --font-serif: var(--font-serif-brand), var(--font-serif-ko), ui-serif, Georgia, serif;
```
(라틴은 Noto Serif KR, 한글은 Nanum Myeongjo가 커버. 폴백 순서상 라틴 글리프는 앞 폰트, 한글은 다음 폰트가 담당.)

- [ ] **Step 4: 검증** — `bunx tsc --noEmit` 클린, `bun run build` 성공. dev에서 404 페이지 "페이지를 찾을 수 없어요"가 세리프(명조)로 렌더되는지 오케스트레이터 스크린샷 단계에서 확인.

- [ ] **Step 5: Commit**
```bash
git add src/fonts src/app/fonts.ts src/app/globals.css src/app/layout.tsx
git commit -m "feat(brand): self-host Nanum Myeongjo for Korean serif headlines"
```
